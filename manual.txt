기능 설명

1. 회원 관련 기능

A. 회원 가입 : 회원 정보를 입력하여 가입함. 
ID, PW, ZIPCODE, NAME, PHONENUM, CITY, GENDER, AGE, JOB 정보를 기입하는 란을 구현하였고 필수정보인 ID, PW, 주소(ZIPCODE, CITY), PHONENUM가 입력되지 않으면 경고 메세지를 출력하고 처음부터 다시 입력받는다.

B. 회원 정보 수정 : 회원의 정보를 수정. (아이디의 경우, 변경 불가)
ID 를 제외한 회원정보를 수정할수있다. 모두 수정하지않아도 업데이트 되도록 구현하였다.

C. 비밀 번호 수정 : 회원의 비밀번호를 수정. 
ID 를 입력받고 새로운 패스워드를 두번 입력받아 두 번 모두 비밀번호가 일치하면 업데이트 되도록 구현하였다. 

D. 로그인 : 회원의 아이디와 비밀번호를 입력하고 쇼핑몰에 접속. 
ID , PW 둘 모두 입력되어야 로그인 검사를 하도록 하였다. db에서 ID, PW가 서로 일치하면 메인페이지로 넘어가고 불일치하면 다시 로그인한다.

E. 관리자 계정
고객계정과 관리자 계정을 따로 만들어 관리자 계정으로 로그인시에 메인페이지는 같지만 다른 기능을 제공하도록 하였다. 



2. 물품 관련 기능 

A. 회원은 로그인 이후 카테고리 별로 물품을 “구경”하는 것이 가능. 
(카테고리를 선택하여 들어가면 물품 리스트가 보이도록 함.)
메인페이지에서 기능을 구현하였다. 왼쪽상단 메뉴를 클릭시에 카테고리바가 나오며 대분류, 소분류를 보여준다. 소분류를 클릭시에 그 소분류에 포함되는 아이템 리스트를 보여준다.


B. 회원은 검색을 통해 특정 물품을 찾아보는 것이 가능. 
메인페이지에서 검색칸을 통해 구현하였다. 디비에 연결된 상품리스트를 보여주며 상품이름 검색시에 그에 맞는 상품을 보여준다. 

C. 물품을 선택하면 물품의 정보를 확인할 수 있으며, 해당 물품을 장바구니에 추가할 수 있음. 
물품을 클릭하면 물품의 상세페이지로 넘어가서 정보를 확인할수있다. 오른쪽 하단 버튼으로 장바구니에 추가할수있고 왼쪽 하단 버튼으로 메인페이지로 다시 돌아갈수있다.

D. 구매하고자 하는 개수 보다 재고가 부족할 시, 구매를 하지 못하도록 함. 
재고가 부족할시에 장바구니에 담을 수는 있지만 구매를 할수는 없다.


3. 장바구니 관련 기능
A. 장바구니에 담긴 제품은 로그아웃, 종료 등을 한 뒤에도 유지됨. 
장바구니에 담은 제품은 디비에 연결되어 장바구니에 담긴 아이템 리스트를 따로 저장하게된다. 로그아웃, 종료 후에도 유지된다.

4. 구매 관련 기능 

A. 장바구니에서만 구매가 가능. 
장바구니에서 구매할 제품을 체크하여 order 한다.

B. 회원은 자신의 과거 구매 내역을 확인할 수 있음. 
구매후에 구매내역을 확인할 수 있고 체크하여 주문취소를 할수있다.

5. 관리자 기능 - 관리자 계정으로 접속하였을 때만 사용이 가능한 기능. 

A. 물품을 주문하여 재고를 증가시킬 수 있음. 
(편의상 해당 기능을 실행하면 즉시 재고가 증가하는 것으로 함.)
물품 이름, 늘릴 재고량, 매장을 입력하여 재고를 증가시킬수있다.

B. 재고가 모자란 아이템 리스트를 확인할 수 있음. 
매장별 재고가 2이하인 아이템 리스트를 디비에서 찾아 보여준다.

C. 전체 매출, 월별 매출, 일별 매출 등을 확인할 수 있음.
전체매출을 볼수있고 년도별 클릭을 이용해 월별매출을 볼수있다.
월별 클릭을 이용해 일별 매출을 확인할수있다.

유의사항
Mysql, JDBC등 의 환경 세팅과 연결이 되어있음 을 전제조건으로 한다.
